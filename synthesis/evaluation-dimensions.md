# Evaluation Dimensions

This file tracks the evaluation dimensions used across LLM-based code review papers.

## Core Dimensions

- Technical correctness
- Relevance to the code change
- Usefulness
- Actionability
- Specificity
- Hallucination / unsupported claims
- False positives
- False negatives
- Preservation of useful comments
- Review coverage
- Human annotation quality
- Computational cost
- Latency
- Operational complexity
- Human verification cost
- Workflow impact
- Developer trust

## Working Note

A key goal is to separate dimensions that are often collapsed together in current papers, especially correctness, usefulness, actionability, and developer preference.

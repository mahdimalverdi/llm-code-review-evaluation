# Research Gap

This file collects the emerging research gap.

## Draft Gap Statement

The literature on LLM-based code review increasingly recognizes that simple text-similarity metrics and offline benchmarks are insufficient. Recent work introduces richer rubrics, hallucination judges, production evaluations, and PR-level benchmarks. However, existing studies still lack a unified framework that jointly evaluates:

- context quality,
- problematic comment types,
- preservation of useful feedback,
- filtering or mitigation decisions,
- human annotation validity,
- and cost/workflow trade-offs.

## Possible Contribution

A taxonomy and trade-off-aware evaluation framework for LLM-generated code review comments, with special attention to context quality and the consequences of filtering or suppressing generated comments.
